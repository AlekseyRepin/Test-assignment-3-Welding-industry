# Test-assignment-3-Welding-industry
Тестовое задание №3 Сварочное производство

Задание 1
Для выполнения задания нужно создать пустую базу.
Создать справочники:
1)	Блоки
Реквизиты:
Код – Номер блока из Файла DemoBD
Наименование не используется
2)	Линии
Реквизиты:
Код не используется
Наименование – номер линии из Файла DemoBD
3)	Сварщики
Реквизиты:
Код не используется
Наименование – Имя сварщика из Файла DemoBD
4)	Изометрии
Реквизиты:
Код не используется
Наименование – Изометрия из Файла DemoBD
Блок – Ссылка на справочник блоки
Линия – Ссылка на справочник линии
Ревизия – Число из Файла DemoBD
Готовность – Число (процент от 0 до 100)
Табличная часть:
Номер стыка – Число
Дата Сварки – Дата
Сварщик – Ссылка на справочник сварщики
Заполнение базы на Ваш выбор:
1)	В форме списка справочника Изометрии добавить кнопку «Импорт», при нажатии на кнопку открывается окно выбора файла для импорта. Файл передается на сервер, для импорта.
Данные импортируются в справочник изометрии. Первые четыре колонки в файле – это реквизиты справочника, остальные три – табличная часть.
При этом для колонок Блок, Линия и Сварщик ищется ссылка на соответствующий справочник. Если нужного элемента в справочнике нет, то создается новый.
2)	Ручное заполнение
 При сохранении справочника Изометрии, происходит расчет готовности. Стык готов, если указана дата сварки. Например, у изометрии всего 8 стыков, с датами сварки (готовые) - 4 стыка, тогда процент равен 4/8*100 = 50%. Процент готовности записывается в реквизит «Готовность».
В форме списка справочника Изометрий, изометрии с готовностью 100% выделены цветом или шрифтом. 

Задание 2
Для выполнения задания нужно использовать 1Cv8_Задание2.dt.
Дано:
Справочник Блоки.
Используется предопределенный реквизит – Код.
Справочник Сварщики. 
	Наименование – ФИО.
	Код – Клеймо сварщик.
	Руководитель – ссылка на этот же справочник Сварщики.
	Блок – ссылка на справочник Блоки.
	КоличествоСтыков – сколько всего стыков сварил сварщик/руководитель.

Надо изменить текст запроса кнопок «Запрос1», «Запрос2» и т.д., так что бы
Запрос 1 показывал сварщиков, сваривших больше стыков, чем их руководитель.
Запрос 2 показывал список сварщиков, сваривших максимальное количество стыков в своем блоке. Не включая руководителей.
Запрос 3 показывал список блоков, где ответственных за блок (сварщики + руководитель) не более трех человек.
Запрос 4 показывал список сварщиков имеющего руководителя в другом блоке.
Запрос 5 показывал блок с максимальным суммарным количеством стыков.

Кнопки расположены в «Общие команды».

 
Задание 3
Написать алгоритм преобразования единиц измерения длины.
 
На форме «Единицы измерения» при нажатии на стрелку, число должно преобразовываться в выбранные единицы измерения. Например, исходные 15 см при преобразовании в метры должны стать 0,15
Справочник Единицы измерения состоит только из предопределенных элементов, на них можно полагаться при написании алгоритма.
